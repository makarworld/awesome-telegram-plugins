## Кратко

Версия **1.7.1** — доработка undo/cutify поверх **1.7.0**: ручная омилка из меню, стек версий, защита ссылок и @mention, исправление сохранения оригинала при повторной омилке.

---

## ✨ Добавлено

### Cutify — «Омилить ✨»

- Пункт **MESSAGE_CONTEXT_MENU** (priority 9): ручная омилка своего сообщения.
- Работает поверх уже милого текста — каждое нажатие добавляет версию в стек.
- Иконка: `menu_premium_effects`.

### Undo 2.0 — «Отменить омиление»

- Переименован пункт undo; иконка `chats_undo`.
- **Стек версий** вместо одной пары оригинал/милое: `[оригинал, милое₁, милое₂, …]`.
- Шаг назад по истории; на оригинале — переключение **оригинал ↔ первое милое**.
- Лимит кэша: **4096×4 = 16 384 символа** суммарно (сообщения evict целиком, не режутся).

### Защита фрагментов при омилке

Не трансформируются:

- Entity: Url, TextUrl, Mention, MentionName, Phone, Email, BotCommand, Code, Pre.
- Plain-текст (regex): `http(s)://`, `www.`, `t.me/`, `tg://`, `@username`, `tg://user?id=…`, телефоны `+…`.

---

## 🐛 Исправлено

- **Оригинал терялся** при «отправил → омилить вручную»: запись undo не находилась по `(dialog_id, msg_id)` и перезаписывалась с уже милым текстом как «оригиналом».
- Поиск записи: алиасы `dialog_id`, fallback по `msg_id` и тексту, `dialog_id` из ответа API при отправке.
- Undo больше не восстанавливал чужую версию из‑за fallback по неточному ключу.

## ✦ Вердикт: ✅ Безопасно

**Назначение:** локальная трансформация исходящих текстовых сообщений пользователя (UwU, эмодзи, рамки и т.д.), опциональный фильтр чатов, undo/cutify через редактирование **своих** сообщений.

---

## ❏ Анализ

### Обработка данных

- Весь текст обрабатывается **на устройстве**; нет `requests`, `urllib`, сокетов, webhook'ов.
- Кэш undo (`_undo_cache`) — in-memory dict в процессе клиента; не пишется на диск отдельно от настроек плагина.
- Настройки и списки чатов — JSON в `set_setting` / `get_setting` (стандартный API exteraGram).

### Хуки и API

| Хук / API | Использование | Риск |
|-----------|---------------|------|
| `on_send_message_hook` | MODIFY исходящего текста/caption | Низкий — только свои отправки |
| `post_request_hook` | Привязка undo к `msg_id` после send | Низкий — чтение ответа API |
| `MESSAGE_CONTEXT_MENU` | Cutify / Undo на своих сообщениях | Низкий — проверка `_is_outgoing_message` |
| `edit_message` | Восстановление/омилка через edit | Низкий — только вызванные пользователем действия |
| `CHAT_ACTION_MENU` / `DRAWER_MENU` | Настройки, whitelist/blacklist | Низкий |
| `MethodHook` (ChatActivity) | Обновление текста кнопки меню чата | Низкий — UI only |

### Новое в 1.7.1 (доп. проверка)

- **Cutify:** повторная трансформация текста + `edit_message` — без новых сетевых или файловых операций.
- **Стек версий undo:** хранит копии текста/HTML в RAM; лимит 16 КБ снижает DoS по памяти в рамках одной сессии.
- **Защищённые spans:** regex для URL/@/телефонов — только разметка границ, без `eval`/`exec`.
- **Поиск undo по алиасам dialog_id:** не расширяет доступ к чужим данным — только улучшает lookup своих записей.

### Опасные конструкции

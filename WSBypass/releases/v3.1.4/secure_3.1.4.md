❏ Анализ:
• Плагин реализует функциональность обхода блокировок через локальный прокси, что само по себе не является вредоносным. Однако присутствуют риски среднего уровня из-за ослабленной проверки SSL в резервном режиме и потенциальной уязвимости к MITM-атакам. Динамическая загрузка маршрутов и DNS-разрешение требуют дополнительного контроля безопасности.
• Плагин использует стандартные API exteraGram для работы с прокси и настройками.
• Реализована собственная логика WebSocket-подключений к серверам Telegram (web.telegram.org) для обхода блокировок.
• Используется криптография (AES, Ed25519) для защиты трафика, что соответствует безопасности MTProto.
• Присутствует проверка подписи обновлений с использованием Ed25519, что снижает риск подмены.
• Сетевые запросы ограничены известными доменами (th3web.com) и используют HTTPS.
• Локальный прокси работает только на 127.0.0.1, что исключает внешний доступ.
• Отсутствует явная эксфильтрация данных или выполнение произвольного кода.
• Риск связан с динамическим разрешением DNS и подключением к внешним серверам без строгой валидации сертификатов в некоторых случаях.

### Security Review (diff 3.0.5 → 3.1.4)

**Вердикт:** подтверждённых уязвимостей **medium / high / critical** в изменённом коде **не найдено**.

- `tgws_lte_entitled` — только UI-gate; туннель LTE без серверного credential (`hmac` + срок) не поднимается
- `allow_trickle: False` для `lte` — без валидного `X-Cred` туннель недоступен
- Ed25519 auto-update и TLS fallback `CERT_NONE` — логика без изменений vs 3.0.5

**Рекомендации (defense-in-depth, не findings):**

| Область | Рекомендация |
|---------|--------------|
| LTE inline | Allowlist `bot_username == "VPN_Lean_bot"` перед `getInlineBotResults` |
| `tgws_lte_entitled` | Сбрасывать флаг при истечении credential |
| Inline verification | Вернуть `run_on_queue()` для стабильности MTProto с фонового потока |
| TLS fallback | Предупреждение пользователю при `CERT_NONE` на LTE-реле |

# Промпт для vision-анализа emoji-пака

Прикрепите к запросу файл **`pack_sheet.png`** из `output/<pack>/` и вставьте текст ниже.

---

Ты видишь сетку custom emoji из одного Telegram-пака. У каждой ячейки есть номер `#001`, `#002`, … и короткий суффикс `document_id` (последние 6 цифр).

**Задача:** для **каждого** номера опиши смысл эмодзи так, чтобы потом можно было подбирать их по настроению и контексту в чате.

Верни **только валидный JSON** (без markdown, без ```), в формате:

```json
{
  "pack": "SHORT_NAME_ПАКА",
  "items": [
    {
      "index": 1,
      "tags": ["тег1", "тег2", "тег3"],
      "description": "Краткое описание того, что изображено (1–2 предложения, русский)",
      "description_emoji": "😢💧😞",
      "mood": "sad",
      "use_when": ["ситуация1", "ситуация2"]
    }
  ]
}
```

**Правила:**

1. `index` — целое число, как на сетке (`#001` → `1`).
2. `tags` — 3–8 коротких тегов на русском (эмоция, действие, мем, персонаж).
3. `description` — что именно на картинке; для мем-паков укажи отсылку, если узнаёшь.
4. `description_emoji` — то же по смыслу, но **только Unicode-эмодзи** (3–8 штук), без букв и слов.
5. `mood` — одно слово на английском: `happy`, `sad`, `angry`, `neutral`, `ironic`, `love`, `shock`, `tired`, `cringe`, `wholesome` и т.п.
6. `use_when` — 2–5 фраз: в каких репликах/ситуациях уместно вставить это эмодзи.
7. Опиши **все** ячейки с номерами на изображении, без пропусков.
8. Если эмодзи неразборчиво — всё равно дай лучшую гипотезу и пометь в `tags` словом `неуверенно`.

Имя пака для поля `pack`: замените на short name (например `epicdepartmenthopeful_by_fStikBot`).

Сохраните ответ в файл `output/<pack>/labels_by_index.json`, затем:

```powershell
python merge_labels.py --pack epicdepartmenthopeful_by_fStikBot
```
